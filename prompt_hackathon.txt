PROJECT GOAL
------------
Build a production-grade, JWT-secured AI threat detection API that:
  - Detects phishing URLs via BERT ONNX (VitisAI/ROCm/CPU)
  - Detects malicious code via CodeBERT ONNX (VitisAI/ROCm/CPU)
  - Detects behavioral anomalies via MiniLM + Isolation Forest (CPU)
  - Explains threats in 3 tiers: Technical (LIME), User (BART NLI),
    Mentor (hygiene advice via BART + templates)
  - Persists every request to PostgreSQL audit log
  - Caches ML results in Redis with 1-hour TTL + SHA-256 keys
  - Runs all 5 services via podman-compose up
  - Costs ₹0 — 100% free and open-source

SETTINGS REQUIREMENTS (config/settings.py)

  - MODEL_CACHE_DIR, VITIS_CACHE_DIR, LOG_FORMAT, LOG_LEVEL

ONNX RUNTIME ENGINE (ml/onnx_runtime.py)
  - Auto-detect providers at startup: try VitisAI → ROCm → CUDA → CPU
  - VitisAI provider_options: cacheDir + cacheKey (NOT XLNX_ENABLE_CACHE)
  - All inference in ThreadPoolExecutor (never blocks asyncio event loop)
  - ModelTask enum: PHISHING, MALWARE, BEHAVIORAL
  - health() method: returns dict of loaded models + active provider
  - Lazy loading: load model only on first infer() call, then cache session

DOWNLOAD SCRIPT (scripts/download_models.py)
  - CLI via typer: --fp32-only, --log-format (console/json), --force
  - Step 1: Export BERT phishing to FP32 ONNX via Optimum
  - Step 2: Export CodeBERT malware to FP32 ONNX via Optimum
  - Step 3: Export MiniLM to FP32 ONNX via Optimum (NO quantization)
  - Step 4 (AMD hardware only): INT8 quantization via AMD Quark A8W8
    - 200 calibration samples from PhishTank CSV + Tranco top-1M
    - Measure FP32 baseline F1 on 100 eval samples
    - Quantize to INT8
    - Measure INT8 F1, compute delta, enforce ≤ 3% drop
    - Write models_cache/quantization_report.json
  - Step 5: Run a single inference on each model to confirm working
  - AMD Quark wrapped in try/except: missing → FP32-only, log warning
  - Bundled seed data (100 phishing, 100 legit, 50 malicious, 50 clean)
    so the script passes in offline CI without network

CALIBRATION DATA SOURCES (no auth tokens in code)
  - Phishing URLs: PhishTank CSV from phishtank.org (free account)
  - Legitimate URLs: Tranco top-1M from tranco-list.eu (no auth)
  - Malicious code: NIST SARD Juliet Test Suite patterns (public)
  - Clean code: bundled secure coding patterns
  - Fallback: bundled seeds if any network source is unavailable

THREAT RESULT SCHEMA (dataclass, not Pydantic yet)
  @dataclass
  class ThreatResult:
      is_threat: bool
      confidence: float
      severity: str        # CRITICAL/HIGH/MEDIUM/LOW
      model_task: str
      provider_used: str
      latency_ms: float
      cached: bool
      cache_key: str

CACHE KEY DESIGN
  key = SHA-256(f"{task}:{normalized_input}")
  TTL = 3600 seconds (1 hour)
  Normalized: strip whitespace, lowercase for URLs

ISOLATION FOREST DESIGN
  - Fit once at startup on 500 synthetic normal system events
  - Persist fitted model to models_cache/isolation_forest.joblib
  - Load on subsequent startups (skip fit if file exists)
  - MiniLM embedding via ONNX (CPU), then Isolation Forest score
  - anomaly_score → confidence mapping: score < -0.2 = high anomaly

3-TIER OUTPUT SPEC
  Technical : LIME token scores [(token, score), ...] top 10 by |score|
  User      : BART NLI plain English — what the threat is
  Mentor    : BART NLI + template — what the user should do differently
  Only generated when confidence >= 0.70 (avoid LIME cost on low-conf)
  Cached in Redis with same key as the ThreatResult (extend TTL)

POSTGRES SCHEMA
  threat_logs: request_id (UUID PK), timestamp, task, is_threat,
    confidence, severity, provider_used, latency_ms, explanation_json (JSONB)
  api_keys: id (UUID PK), key_hash (VARCHAR 64), scope, last_used
